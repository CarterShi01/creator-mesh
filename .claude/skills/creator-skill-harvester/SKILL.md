# Creator Skill Harvester

Use this skill after completing a meaningful development session.

The goal is to turn expensive learning into reusable skill assets.

## Procedure

1. Review what changed in the session.
2. Identify repeated reasoning patterns, workflows, checks, or decisions.
3. Decide whether any pattern is reusable.
4. If reusable, propose a skill candidate.
5. If not reusable, say no skill should be created.

## Skill Candidate Format

For each candidate, provide:

- Skill name
- Trigger condition
- Problem it solves
- Reusable procedure
- Inputs
- Outputs
- Edge cases
- Suggested location:
  - project skill
  - private CodeSkill repo
  - not worth saving

## Rules

- Do not create the skill automatically unless asked.
- Do not overfit skills to one tiny change.
- Prefer skills that reduce future context cost.
- Prefer skills that can be reused across projects.
- Mention if the pattern should instead become documentation, a script, or a test.
